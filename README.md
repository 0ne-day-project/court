# 프로젝트 개요
본 프로젝트는 하나의 주장(Claim)에 대해 단일 AI의 응답이 아닌, 여러 에이전트 간의 토론 과정을 통해 사실 여부를 판단하는 **멀티 에이전트 시스템(Multi-Agent System)** 구축을 목표로 합니다.

기존의 단일 LLM 기반 응답 방식은 다음과 같은 한계를 가집니다:
* **편향된 답변:** 특정 학습 데이터에 치중된 시각 제공
* **근거 부족:** 논리적 비약이나 출처가 불분명한 정보 생성
* **불확실성 표현 부족:** 할루시네이션(환각)으로 인한 잘못된 정보 전달

이를 해결하기 위해 본 시스템은 서로 다른 관점을 가진 에이전트 간의 상호작용과 논리적 충돌을 활용하여 보다 신뢰도 높은 판단을 생성합니다.

# 노드 구성
* **Supporter (주장 옹호자 - 소희)**
  * 입력된 주장을 사실이라고 가정하고 논리적 근거를 기반으로 주장 강화
  * Tavily Search를 통해 실시간 웹 데이터를 수집하여 설득력 있는 근거 제시
* **Skeptic (회의론자 - 지인)**
  * 입력된 주장을 비판적으로 검토하며 논리적 오류 및 반례 제시
  * 상대 주장의 취약점을 공격하여 논의의 깊이를 확보
* **Fact Checker (팩트 검증자 - 구건희)**
  * Supporter와 Skeptic의 주장을 종합 분석하여 객관적 사실 기준으로 검증
  * 신뢰 가능한 정보와 불확실한 정보를 구분하여 심판에게 전달
* **Judge (최종 판단자 - 우진)**
  * Fact Checker의 결과를 기반으로 최종 결론 도출

## 실험 결과 및 사례 분석

### **[수행 로그]**
```
[시스템 시작] AI 재판이 시작됩니다. 
========================================
🔴 [실행 중] 지인(Skeptic)이 반대 논리를 생성합니다...
🔍 [Skeptic 검색어 생성 중...]
🟢 [실행 중] 소희(Supporter)가 찬성 논리를 생성합니다...
🔍 [검색 중] Tavily로 찬성 근거를 검색합니다...
🔍 [검색 중] Tavily로 반대 근거를 검색합니다 (키워드: 한화이글스 승리 논란)
[실행 중] Fact Checker가 양측 주장을 검증합니다...
⚖️ [실행 중] 판사(Judge)가 1번째 판결을 내립니다...
⚖️ [판사] 탕탕탕! 토론 종결을 선언합니다.

========================================
🎉 [시스템 종료] 최종 판결문이 도착했습니다!
========================================
판결

사건 요지
- 쟁점 문장: “최근 한화이글스는 경기를 우승했다.”
- 쟁점은 문장의 의미(단일 경기의 승리 주장인지, 아니면 시리즈·한국시리즈 최종 우승을 주장하는지)의 명확성이다.

사실관계 요약(검증된 사실)
1. 한화이글스는 전·중반기에 우수한 성적(예: 전반기 52승·2무·33패, 승률 .612 등)과 긍정적 언론평을 받은 것은 사실이다.
2. 한화는 한국시리즈에서 특정 경기(예: 3차전)를 7–3으로 승리한 보도가 존재한다(따라서 ‘최근 경기 승리’는 사실로 확인된다).
3. 그러나 복수 보도에 따르면 한국시리즈 최종 우승(챔피언)은 LG 트윈스로 보도되고 있다. 즉, ‘한국시리즈 우승(시리즈 최종 우승)’을 한화가 차지했다는 주장은 제공 자료들과 충돌한다.

법리적 판단(해석 및 결론)
- 문장이 “경기에서(단일 경기) 우승했다”라는 의미로 쓰였다면: 사실이다(단일 경기 승리는 확인됨). 이 부분은 인정한다.
- 문장이 통상 독자가 이해하는 바대로 “우승”을 시리즈·챔피언(한국시리즈 우승)으로 해석한 것이라면: 제공된 근거들과 모순되므로 허위 또는 오해의 소지가 있다(기각한다).
- 결론적으로 원문(단정적·문맥 불명확한 표현) 그대로의 주장은 의미 해석에 따라 사실성과 허위성이 갈리므로, “명확한 의미 표기가 없는 상태에서의 단정적 진술”은 근거 부족·오해 유발로서 받아들일 수 없다.

판결
- 문장 해석별 결과:
  - 단일 경기 승리를 주장하는 의미: 인정(사실).
  - 시리즈(한국시리즈) 최종 우승을 주장하는 의미: 기각(허위/오해 소지).
- 종합적 판단: 피고(찬성 측)가 제시한 일부 사실(성적·특정 경기 승리)은 타당하나, 이를 근거로 일반인이 통상 해석하는 ‘우승=챔피언’으로 단정한 주장은 근거 부족이므로 받아들일 수 없다. 따라서 논쟁의 최종 승리는 ‘의미의 명확성을 지적한’ 반대 측(지인)에게 돌아간다.

이상으로 본 사건을 종결한다. 종결.
Disconnected from server
```

# 프로젝트 기술 스택
- Framework: LangGraph, LangChain
- Language: Python 3.11+
- LLM: OpenAI GPT-4o
- Search API: Tavily Search


# 프로젝트에서 잘한 점
- 논리적 견제 구조: Supporter와 Skeptic이 서로 다른 검색어를 사용하여 정보를 수집함으로써 단일 시각에 매몰되지 않는 구조를 완성했습니다.
- 실시간 팩트체크 연동: 정적 데이터가 아닌 Tavily Search를 통한 실시간 웹 검색으로 최신 의혹과 해명 자료를 모두 반영했습니다.
- 전문적인 출력 포맷: 판결문 형식을 도입하여 단순 답변이 아닌 '검증 프로세스'를 시각적으로 명확히 보여주었습니다.

# 프로젝트에서 아쉬운 점
- 토큰 소모 및 비용: 에이전트 간 대화가 길어질수록 Context가 누적되어 API 비용이 상승하는 문제가 있었습니다.
- 상태 관리의 복잡성: 에이전트가 늘어남에 따라 데이터가 섞이지 않도록 엄격한 포맷팅이 필요했습니다.
- 배운 점: 기술적으로 뛰어난 프롬프트보다, 에이전트 간의 **'데이터 전달 규약(State Interface)'**을 잘 정의하는 것이 멀티 에이전트 시스템의 핵심임을 배웠습니다.

# 프로젝트 고도화 계획
- Self-Correction 루프: Fact Checker가 근거 부족을 지적하면 다시 보완하는 피드백 루프 추가.
- Memory 기능 도입: 과거 토론 이력을 기억하여 유사한 주제에 대해 더 심도 있는 논의 가능.
- UI 대시보드: 토론 과정과 흐름을 한눈에 볼 수 있는 인터페이스 구축.
